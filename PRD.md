# 제품 요구사항 문서 (PRD): 세방리튬배터리 SCM 시스템

## 1. 개요

본 문서는 '세방리튬배터리 SCM(공급망 관리) 시스템'의 제품 요구사항을 정의합니다. 이 시스템은 리튬 배터리 생산 및 공급과 관련된 전 과정을 효율적으로 관리하고, 각 담당자 간의 원활한 협업을 지원하기 위한 웹 기반 플랫폼입니다. 발주, 생산, 자재, 판매, 출하에 이르는 전체 공급망 프로세스를 통합하여 데이터 가시성을 높이고 운영 효율성을 극대화하는 것을 목표로 합니다.

## 2. 목표

- **프로세스 통합:** 파편화된 공급망 관리 업무를 단일 플랫폼에서 처리하여 데이터의 일관성을 확보하고 중복 작업을 제거합니다.
- **실시간 가시성 확보:** 모든 사용자가 자신의 역할에 맞는 최신 정보를 실시간으로 확인하여 신속하고 정확한 의사결정을 내릴 수 있도록 지원합니다.
- **역할 기반 접근 제어:** 각 담당자의 역할(발주사, 생산관리자, 창고관리자, 자재관리자)에 따라 명확하게 권한을 분리하여 정보 보안을 강화하고 업무 효율성을 높입니다.
- **운영 효율성 증대:** 계획(발주/판매/생산)부터 실행(생산/자재/출하/배차)까지의 과정을 자동화하고 최적화하여 리드 타임을 단축하고 비용을 절감합니다.

## 3. 기술 스택 및 환경 (Technology Stack)

안정적이고 확장 가능한 시스템 구축을 위해 다음과 같은 최신 웹 기술 스택을 사용합니다.

- **Frontend:** Next.js (React Framework), Tailwind CSS (Styling), Lucide React (Icons)
- **Backend:** Node.js (API Server) 또는 Supabase (Backend-as-a-Service)
- **Database:** PostgreSQL (Relational Database)
- **Deployment:** Vercel (Frontend), AWS or Supabase (Backend/DB)
- **Authentication:** Role-Based Access Control (RBAC) with JWT or Supabase Auth

## 4. 사용자 및 역할 (User Roles)

시스템은 다음과 같은 주요 사용자 역할로 구성되며, 각 역할은 특정 기능에 대한 접근 권한을 가집니다.

- **발주사 (Ordering Company/Client):**
  - 제품을 주문하고 납기를 관리하는 외부 또는 내부 고객입니다.
  - **주요 관심사:** 주문 현황, 납기 일정, 출하 상태 추적.
- **생산관리자 (Production Manager):**
  - 제품 생산 계획을 수립하고 실제 생산 실적을 관리합니다.
  - **주요 관심사:** 생산 목표 달성률, 생산 효율성, 생산 실적 관리.
- **자재관리자 (Materials Manager):**
  - 생산 계획에 필요한 원자재 및 부자재의 재고를 관리하고 입출고를 담당합니다.
  - **주요 관심사:** 자재 재고 수준, 자재 부족 예방, 입출고 정확성, BOM 기반 소요량 예측.
- **창고관리자 (Warehouse Manager):**
  - 판매 계획을 수립하고, 생산된 완제품의 출하 및 배차를 총괄합니다.
  - **주요 관심사:** 완제품 재고 관리, 정확한 출하, 효율적인 배차.
- **시스템 관리자 (Admin):** (신규 추가)
  - 사용자 계정 생성 및 권한 관리, 기준 정보(품목, BOM 등) 관리.

## 5. 기준 정보 관리 (Master Data Management)

시스템 운영의 기반이 되는 데이터 관리 기능입니다.

- **품목 관리 (Product/Item Master):**
  - 배터리 모델명, 규격, 단가, 품목 유형(완제품/원자재/부자재) 등록 및 수정.
- **BOM (Bill of Materials) 관리:**
  - 완제품 1단위를 생산하는 데 필요한 원자재/부자재의 종류와 소요량 정의.
- **거래처 관리 (Partner Management):**
  - 발주사(고객) 및 자재 공급사(Vendor) 정보 등록 및 관리.

## 6. 주요 기능 (Features)

- **대시보드 (`/`):**
  - 모든 사용자가 공통으로 접근하는 메인 페이지.
  - SCM 전체 현황을 요약한 주요 지표(KPI) 및 알림을 시각적으로 제공. (예: 주문 현황, 생산 진행률, 재고 수준 등)

- **발주 계획 (`/orders`):**
  - **사용자:** 발주사, 영업팀
  - 신규 발주를 등록하고, 기존 발주 내역의 상태(접수, 생산중, 출하완료 등)를 조회.
  - **상태 흐름:** 접수 -> (승인/반려) -> 생산대기 -> 생산중 -> 생산완료 -> 출하대기 -> 출하중 -> 인수완료

- **판매 계획 (`/sales`):**
  - **사용자:** 창고관리자, 영업팀
  - 시장 수요 예측 및 영업 목표에 기반하여 완제품 판매 계획을 수립.

- **생산 계획 (`/production`):**
  - **사용자:** 생산관리자
  - 발주 계획과 판매 계획을 기반으로 주간/월간 생산 계획을 수립하고 생산 라인에 작업을 할당.
  - 자재 가용성 체크 기능 포함.

- **생산 실적 (`/production-performance`):**
  - **사용자:** 생산관리자
  - 일별/주별 생산 실적(생산량, 불량률 등)을 등록하고 계획 대비 실적을 분석.

- **자재 관리 (`/materials`):**
  - **사용자:** 자재관리자
  - 생산 계획(BOM 기반)에 따른 자재 소요량을 예측하고 현재 재고를 관리.
  - 자재 입출고 내역 기록 및 안전 재고 알림.

- **출하 지시 (`/shipments`):**
  - **사용자:** 창고관리자
  - 발주사의 주문에 따라 출하할 제품, 수량, 일정을 지정하고 출하 지시를 생성.
  - 명세서(Packing List) 출력 기능.

- **배차 관리 (`/dispatch`):**
  - **사용자:** 창고관리자
  - 출하 지시에 따라 운송 차량을 배정하고 운송 상태를 추적.

## 7. 사용자 여정 (User Journey)

### 여정 1: 신규 배터리 주문 및 생산, 출하 프로세스

1.  **[발주사]**
    - 시스템에 로그인하여 '발주 계획' 메뉴로 이동합니다.
    - 필요한 배터리 모델과 수량, 희망 납기를 입력하여 신규 발주를 등록합니다.
    - 대시보드 또는 '발주 계획' 메뉴에서 자신의 주문이 '접수' 상태인지 주기적으로 확인합니다.

2.  **[생산관리자]**
    - 신규 발주가 등록되었다는 알림을 확인하고 '생산 계획' 메뉴로 이동합니다.
    - 발주량과 현재 재고를 고려하여 생산 계획을 수립하고 작업 지시를 내립니다.
    - 생산이 진행됨에 따라 '생산 실적' 메뉴에 일일 생산량과 특이사항을 기록합니다.
    - 생산이 완료되면 시스템에 '생산 완료' 상태를 업데이트합니다.

3.  **[자재관리자]**
    - 수립된 '생산 계획'을 확인하고 BOM을 참조하여 필요한 원자재 소요량을 자동 계산/확인합니다.
    - '자재 관리' 메뉴에서 현재 자재 재고를 확인하고, 부족할 경우 추가 발주를 진행합니다.
    - 생산 라인에 자재가 투입될 때마다 출고 내역을 시스템에 기록합니다.

4.  **[창고관리자]**
    - '생산 완료'된 주문을 확인하고 '출하 지시' 메뉴로 이동합니다.
    - 해당 주문에 대한 출하 정보를 확인하고 출하 지시를 생성합니다.
    - '배차 관리' 메뉴에서 출하 지시에 가장 적합한 운송 차량을 배정하고, 운송 기사에게 정보를 전달합니다.
    - 차량이 배정되고 제품이 상차되면, '출하 중'으로 상태를 변경합니다.

5.  **[발주사]**
    - 자신의 주문 상태가 '출하 중'으로 변경된 것을 확인하고, 예상 도착 시간을 파악합니다.
    - 제품을 수령한 후 시스템에서 '인수 완료' 처리를 합니다.

## 8. 비기능적 요구사항

- **보안:**
  - 역할 기반 접근 제어(RBAC)를 통해 인가되지 않은 정보 접근을 엄격히 차단해야 합니다.
  - 모든 데이터 통신은 SSL/TLS를 통해 암호화되어야 합니다.
- **성능:**
  - 모든 페이지는 3초 이내에 로드되어야 합니다.
  - 대용량 데이터 조회 시에도 시스템 응답성이 저하되지 않아야 합니다. (예: 페이지네이션, 필터링 기능 제공)
- **안정성:**
  - 시스템은 연중무휴 24/7 운영을 목표로 하며, 99.9%의 가용성을 보장해야 합니다.
- **사용성:**
  - 사용자가 별도의 교육 없이도 직관적으로 사용할 수 있는 UI/UX를 제공해야 합니다.
  - **모바일/태블릿 지원:** 현장 관리자(생산, 창고)를 위해 반응형 디자인을 적용해야 합니다.
- **데이터 활용:**
  - 주요 데이터(주문 내역, 재고 현황 등)는 **Excel 다운로드** 기능을 제공해야 합니다.
